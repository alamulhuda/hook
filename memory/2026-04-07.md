# 2026-04-07 - Session with Dan

## What happened

- Dan asked to fix Docker container `arabica-db` exiting.
- Root cause was a Compose mismatch:
  - `.env` still uses `DB_HOST=db`
  - `docker-compose.yml` had `db` behind `profiles: ["local"]`, so normal `docker compose up -d` did not start the database service
  - `arabica-app` and `arabica-queue` started without a resolvable `db` hostname

## Fix applied

- Updated `docker-compose.yml`:
  - removed `profiles: ["local"]` from `db`
  - added `depends_on: [db]` to `app`
  - added `depends_on: [db]` to `queue`
  - replaced MySQL command flags:
    - removed invalid `--tls-version=''`
    - changed deprecated `--innodb-log-file-size=48M` to `--innodb-redo-log-capacity=100663296`

## Verification

- Ran `docker compose up -d db app queue web`
- Verified `arabica-db` is up on `127.0.0.1:3307`
- Verified Laravel DB access with `docker exec arabica-app php artisan migrate:status --no-interaction`
- Restarted `arabica-queue` and confirmed no fresh DB resolution errors
